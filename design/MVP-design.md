# 羽毛球装备记录小程序 MVP 需求说明

## 一、项目概述

- 项目名称（暂定）：羽记本（可修改）
- 技术栈：
  - 前端：uni-app + Vue3（`<script setup>`）
  - 目标运行平台：微信小程序
  - 数据存储：**本地存储**（使用 `uni.setStorage` / `uni.getStorage`），暂不接云端/后端服务
- 目标用户：个人羽毛球爱好者
- MVP 核心目标：提供一个简单、易用的界面来记录和管理个人羽毛球装备（球拍、球鞋、球包等），支持新增、查看、编辑、删除。

---

## 二、功能模块

### 1. 装备列表页（首页）

#### 1.1 功能描述
- 展示当前用户已记录的所有装备列表。
- 支持按装备类型简单分组或标注（如球拍、球鞋、球包、其他）。
- 每条装备显示核心信息，点击进入详情页。
- 提供入口跳转到「新增装备」页面。

#### 1.2 展示字段（列表项）
每个装备列表项至少展示：
- 品牌 + 型号（组合为名称，如 `Yonex Astrox 88D`）
- 类型（球拍 / 球鞋 / 球包 / 其他）
- 购买日期（如：`购于：2023-10-01`）
- 使用状态：显示为「使用中」或「已淘汰」

#### 1.3 交互需求
- 点击某条装备 → 跳转到装备详情页，并携带装备 ID。
- 点击「添加装备」按钮 → 跳转到新增装备页。
- 空列表状态：
  - 当没有任何装备记录时，显示一条引导文案，例如：
    - 「还没有记录任何装备，点击下方按钮开始添加吧。」

---

### 2. 装备详情页

#### 2.1 功能描述
- 展示某一件装备的完整信息。
- 支持从详情页进入编辑模式。
- 支持删除该装备（删除后返回列表页）。

#### 2.2 展示字段（详情）
- 品牌（如：Yonex）
- 型号（如：Astrox 88D）
- 类型：球拍 / 球鞋 / 球包 / 其他
- 价格（整数或小数，单位：元）
- 购买日期（格式：`YYYY-MM-DD`）
- 使用状态：
  - 使用中 / 已淘汰
- 使用时间
- 备注（可选字段，用于记录个人感受或说明）

#### 2.3 交互需求
- 从列表页携带装备 ID 到详情页，详情页根据 ID 从本地存储中读取对应数据。
- 「编辑」按钮：
  - 点击后跳转到编辑装备页（可以复用新增页的表单逻辑），并带上装备 ID。
- 「删除」按钮：
  - 弹出确认对话框：
    - 文案示例：「确认删除这件装备记录吗？此操作不可恢复。」
  - 确认后删除该装备记录，更新本地存储，并返回列表页。

---

### 3. 新增/编辑装备页（统一表单页）

> 同一页面根据是否传入 `id` 参数区分「新增」或「编辑」。

#### 3.1 功能描述
- 提供一个表单，用于：
  - 新增一件装备；
  - 编辑已有装备的信息。
- 表单提交后保存到本地存储，再返回列表页。

#### 3.2 表单字段定义

字段及类型要求：

1. **品牌（brand）**
   - 类型：字符串
   - 必填：是
   - 示例：Yonex

2. **型号（model）**
   - 类型：字符串
   - 必填：是
   - 示例：Astrox 88D

3. **类型（type）**
   - 类型：枚举字符串
   - 必填：是
   - 可选值：
     - `racket`：球拍
     - `shoes`：球鞋
     - `bag`：球包
     - `other`：其他
   - 表单中采用下拉选择或单选按钮展示对应的中文标签。

4. **价格（price）**
   - 类型：数字（可以用文本输入，内部转为数字）
   - 必填：否（允许为空）
   - 约束：> 0（如非空）
   - 单位：人民币（元）

5. **购买日期（buyDate）**
   - 类型：字符串（格式 `YYYY-MM-DD`）
   - 必填：否（如用户不记得可以留空）
   - 使用日期选择器组件（如 `uni-datetime-picker` 或原生日期选择控件）。

6. **使用状态（retired）**
   - 类型：布尔
   - 必填：是
   - 含义：
     - `false` → 使用中
     - `true` → 已淘汰
   - 表单中使用开关或单选按钮显示。

7. **备注（note）**
   - 类型：字符串
   - 必填：否
   - 文本多行输入框，用于记录个人感受、故事等。

#### 3.3 新增逻辑

- 初始表单为空或带默认值：
  - 类型默认可选 `racket`（球拍），也可留空。
  - 使用状态默认：使用中（`retired = false`）。
- 提交时：
  - 生成一个唯一 ID（例如使用当前时间戳 `Date.now()`）。
  - 将新装备对象插入本地存储中保存的装备列表数组。
  - 成功保存后：
    - 提示「保存成功」；
    - 返回装备列表页，并刷新列表。

#### 3.4 编辑逻辑

- 从详情页跳入编辑页时，携带 `id` 参数。
- 编辑页初始化时：
  - 根据 ID 从本地存储中的装备列表数组中找到该装备；
  - 将其字段值填充到表单中。
- 提交时：
  - 替换列表中对应 ID 的装备数据；
  - 更新本地存储；
  - 成功保存后提示「更新成功」，返回详情页或者列表页（具体实现可自行选择）。

#### 3.5 表单校验需求（基础版）

- 必填项校验：
  - 品牌、型号、类型必填；
  - 价格非必填，但如填写需为正数。
- 校验失败时：
  - 通过 `uni.showToast` 提示具体错误，如：
    - 「请填写品牌」
    - 「价格须为正数」

## 四、数据存储设计（本地存储）

### 4.1 存储键名

- 使用固定键名：`equipList`
- 类型：JSON 字符串表示的装备数组。

### 4.2 装备数据结构

单条装备对象示例（JSON）：

```json
{
  "id": 1690000000000,
  "brand": "Yonex",
  "model": "Astrox 88D",
  "type": "racket",
  "price": 1200,
  "buyDate": "2023-10-01",
  "usage":"50",//使用时间，单位是天数
  "retired": false,
  "note": "第一次打团体赛用的拍，手感偏重但杀球爽。"
}
```
## 5 路由与页面结构（建议）
### 5.1 页面列表（uni-app）
- pages/equip-list/equip-list：装备列表页（首页）
- pages/equip-detail/equip-detail：装备详情页
- pages/equip-edit/equip-edit：新增/编辑装备页
### 5.2 路由跳转约定
- 列表 → 详情：
uni.navigateTo({ url: '/pages/equip-detail/equip-detail?id=EQUIP_ID' })
- 列表 → 新增：
uni.navigateTo({ url: '/pages/equip-edit/equip-edit' })
- 详情 → 编辑：
uni.navigateTo({ url: '/pages/equip-edit/equip-edit?id=EQUIP_ID' })

## 6 UI 与交互（简要要求）
- 整体风格：简洁、以内容为主（不强制特定配色，仅要求清晰易读）。
- 列表项：使用简单卡片样式，区分装备条目。
- 表单控件：使用 uni-app 提供的基础表单组件 组件库路径：uni_modules/
- 反馈： 使用 uni.showToast 提示保存成功、删除成功或表单错误。

## 7 非功能性需求（MVP）
- 代码结构：
使用 Vue3 script setup 语法。
尽量将数据读写封装成简单函数以便后续扩展。
- 兼容性：
主要目标为微信小程序端。
H5 端可运行但不强制优化（MVP 阶段可选）。
